## Commit Message Guidelines##
Clean well-written descriptive commit messages improve understanding and maintain historical records in projects . Commit should reflect change that' implemented..

Commit messages use the following structure `type( scope: brief description)`.. `Type` represents a categorization of commit, `Fix Bug, Add Features or Documentation`. `Scope` describes the area affected . The description is succinct description. 

Examples include `feat: adding new login authentication`, `fix`: resolves memory leaking issue `. Follow conventions to ensure maintain consistent history .
