## Getting Started

To get started, you will need a compatible development environment (e g., a working version of Python and relevant dependencies). Please make sure you have Git installed, which you will use for fetching, branching and contributing code - the project is actively maintained using Git.

Before diving in to contribute, fork the repository on GitHub (or your preferred Git hosting platform). Once forked, clone your fork to your local machine using `git clone <your_fork_url>`.

After you clone the project, you' will need to create a new branch for your contributions - this helps ensure you don't break the `main` development path by directly committing. A good naming scheme is to use something like `feature/your-new-feature or fix/bug-number`. For this use: `git checkout -b feature/new-feature` .

Install any project dependencies listed in ` requirements.txt`. You can typically install dependencies using a command like `pip install -r requirements.txt` or equivalent for your chosen packaging system - make sure your Python installation works!
